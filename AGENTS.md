# Agents

## What this repo is
Hexlet educational project: a TypeSpec API contract for a Calendar Booking system with a React frontend.

## Structure
- `typespec/` — TypeSpec API contract (single source of truth)
- `frontend/` — Vite + React + TypeScript + Mantine UI

## Key files
- `typespec/main.tsp` — API models, routes, request bodies
- `typespec/tspconfig.yaml` — emitter config (OpenAPI 3.0)
- `typespec/tsp-output/openapi/openapi.yaml` — generated OpenAPI spec
- `frontend/vite.config.ts` — Vite config with proxy to Prism (port 4010)
- `frontend/src/main.tsx` — frontend entry point

## Commands

### TypeSpec
```bash
npm install                     # install TypeSpec compiler (in typespec/)
npx tsp compile main.tsp        # regenerate OpenAPI output
```

### Frontend
```bash
cd frontend && npm install      # install frontend deps
npm run dev                     # start Vite dev server (port 5173)
npm run build                   # typecheck + production build
npm run lint                    # ESLint
```

### API mock (Prism)
```bash
npx @stoplight/prism-cli mock typespec/tsp-output/openapi/openapi.yaml --port 4010
```
Vite proxies `/api/*` to Prism automatically.

### Full dev workflow
In two terminals:
1. `npx @stoplight/prism-cli mock typespec/tsp-output/openapi/openapi.yaml --port 4010`
2. `cd frontend && npm run dev`

## Rules
- Do not edit `.github/workflows/hexlet-check.yml` — auto-generated by Hexlet.
- After editing `typespec/main.tsp`, regenerate OpenAPI output before committing.
- Two API interfaces: `Admin` (`/api/...`) and `Public` (`/api/public/...`).
- Prism is not installed as a dependency — run via `npx`.
